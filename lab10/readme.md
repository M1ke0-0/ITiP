### 1. Цель работы
Изучение принципов работы систем автоматической сборки проектов на примере Gradle. Знакомство с архитектурой многомодульных проектов, управлением внешними зависимостями и созданием пользовательских задач (tasks).

### 2. Задание
1.  Создать многомодульный проект на Gradle.
2.  Реализовать вспомогательный модуль `string-utils` с использованием внешней библиотеки `Apache Commons Lang`.
3.  Разработать основное приложение, которое использует возможности модуля `string-utils` и выводит информацию о сборке (Build Passport).
4.  Настроить автоматическую генерацию "паспорта сборки" при каждой компиляции.
5.  Обеспечить совместимость проекта с актуальной версией Gradle (9.x).

### 3. Описание реализации

#### Структура проекта
Проект разделен на два модуля:
*   **lab10 (root)**: Основное приложение.
*   **string-utils**: Библиотека для обработки строк.

#### Конфигурация сборки (build.gradle.kts)
Основные настройки включают:
*   Использование плагина `com.gradleup.shadow:8.3.5` для сборки переносимых JAR-файлов.
*   Настройка `JavaExec` для возможности интерактивного ввода данных пользователем (`standardInput = System.in`).
*   Реализация задачи `generateBuildInfo`, которая извлекает текущий хеш коммита из Git и сохраняет его вместе с системными данными в файл `.properties`.

#### Ключевой код
**Main.java:**
При запуске программа считывает ресурсы, генерируемые Gradle, и выводит подробный паспорт сборки, после чего переходит к интерактивному режиму.

**StringProcessor.java:**
Использует метод `StringUtils.reverse` из Apache Commons для реализации бизнес-логики.
