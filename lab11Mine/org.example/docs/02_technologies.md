# Используемые технологии

## Spring Boot 4.0.5
Фреймворк для быстрого создания Java-приложений. Автоматически настраивает компоненты через `@SpringBootApplication`.

## Spring Data JPA
Абстракция над JPA (Java Persistence API). Позволяет работать с БД через интерфейсы репозиториев без написания SQL вручную.

## Hibernate 7.2.7
ORM (Object-Relational Mapping) — реализация JPA. Автоматически создаёт и изменяет таблицы в БД (`ddl-auto=update`), генерирует SQL по Java-объектам.

## PostgreSQL 18
Реляционная база данных. Хранит данные в таблицах `users` и `notifications`.

## Lombok
Библиотека для генерации шаблонного кода:
| Аннотация | Что генерирует |
|---|---|
| `@Getter` / `@Setter` | Геттеры и сеттеры |
| `@NoArgsConstructor` | Пустой конструктор |
| `@AllArgsConstructor` | Конструктор со всеми полями |
| `@Builder` | Паттерн строителя |
| `@RequiredArgsConstructor` | Конструктор для `final` полей |

## Bean Validation (Jakarta Validation)
Валидация входных данных через аннотации на полях DTO:
| Аннотация | Назначение |
|---|---|
| `@NotBlank` | Строка не null и не пустая |
| `@Email` | Корректный формат email |
| `@Pattern` | Соответствие регулярному выражению |
| `@Size` | Ограничение длины строки |
| `@NotNull` | Значение не null |

`@Valid` в контроллере запускает валидацию. При нарушении возвращается `400 Bad Request`.

## HikariCP
Пул соединений с БД. Управляет подключениями к PostgreSQL эффективно.

## Spring DevTools
Автоматически перезапускает приложение при изменении файлов во время разработки.
