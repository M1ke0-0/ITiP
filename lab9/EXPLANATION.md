# Лабораторная работа №9: Инструмент сборки Maven

## Описание проекта
Этот проект демонстрирует использование инструмента автоматизации сборки Apache Maven. 
В проекте реализована иерархия классов "Животные" (Вариант 1):
- **Animal**: Абстрактный базовый класс с общими свойствами (имя, возраст) и методами.
- **Cat, Parrot, Fish**: Классы-наследники, реализующие специфическое поведение.
- **Main**: Класс для демонстрации работы (создание объектов, вызов методов).

Проект управляется файлом `pom.xml`, который описывает зависимости (JUnit), настройки компилятора и плагин для запуска приложения.

## Запуск
Для сборки и запуска проекта используйте команду (при установленном Maven):
```bash
mvn clean install exec:java
```

## Контрольные вопросы

1. **Что такое Apache Maven и для чего он используется?**
   Apache Maven — это инструмент для автоматизации сборки проектов, управления зависимостями и документацией. Он используется для стандартизации процесса разработки, упрощения сборки и управления библиотеками.

2. **Как установить Maven на различные операционные системы?**
   - **Windows**: Скачать архив с сайта, распаковать, добавить путь к папке `bin` в переменную среды `PATH`.
   - **Linux**: Использовать пакетный менеджер (например, `sudo apt install maven`).
   - **macOS**: Использовать Homebrew (`brew install maven`).

3. **Какова структура проекта Maven?**
   Стандартная структура:
   - `pom.xml` — в корне.
   - `src/main/java` — исходный код приложения.
   - `src/main/resources` — ресурсы приложения.
   - `src/test/java` — исходный код тестов.
   - `target` — директория для скомпилированных файлов и артефактов.

4. **Что такое POM-файл и какова его роль в проекте Maven?**
   POM (Project Object Model) — это XML-файл (`pom.xml`), содержащий информацию о проекте и конфигурацию сборки (зависимости, плагины, версии). Это основной файл, управляющий проектом.

5. **Какова структура файла POM?**
   Основные элементы: `<project>` (корень), `<modelVersion>`, `<groupId>`, `<artifactId>`, `<version>`, `<packaging>`, `<dependencies>`, `<build>`, `<properties>`.

6. **Что такое зависимости (dependencies) в Maven и как они определяются?**
   Зависимости — это внешние библиотеки (jar-файлы), необходимые проекту. Они определяются в секции `<dependencies>` с указанием `groupId`, `artifactId` и `version`.

7. **Что такое репозиторий Maven и какие виды репозиториев существуют?**
   Репозиторий — это хранилище артефактов (библиотек). Виды:
   - **Локальный**: папка на компьютере разработчика (`~/.m2/repository`).
   - **Центральный**: глобальный репозиторий Maven Central.
   - **Удаленный**: корпоративные или сторонние репозитории.

8. **Как добавить зависимость в проект Maven?**
   Добавить блок `<dependency>` с координатами артефакта (GAV) внутрь тега `<dependencies>` в `pom.xml`.

9. **Что такое плагины в Maven и как они используются?**
   Плагины выполняют конкретные задачи (компиляция, тесты, упаковка). Они объявляются в секции `<build><plugins>`. Примеры: `maven-compiler-plugin`, `maven-surefire-plugin`.

10. **Как создать новый проект Maven с помощью команды Maven?**
    Используя архетипы:
    `mvn archetype:generate -DgroupId=com.example -DartifactId=my-app -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false`

11. **Что такое цели (goals) и фазы (phases) в Maven и в чем их отличие?**
    - **Фаза**: Этап жизненного цикла сборки (например, `compile`, `test`, `package`).
    - **Цель**: Конкретная задача, выполняемая плагином (например, `compiler:compile`). Фаза состоит из одной или нескольких целей.

12. **Как выполнить команду сборки проекта в Maven?**
    Ввести в консоли `mvn [фаза]`, например: `mvn clean install`.

13. **Что такое жизненный цикл сборки (build lifecycle) в Maven?**
    Это последовательность фаз, определяющая процесс сборки. Основные жизненные циклы: `default` (сборка), `clean` (очистка), `site` (создание документации).

14. **Как настроить профили (profiles) в Maven для разных сред?**
    Профили определяются в теге `<profiles>` в `pom.xml`. Они позволяют изменять конфигурацию сборки (например, БД для dev и prod) в зависимости от активного профиля (`mvn package -P production`).

15. **Как управлять версиями зависимостей в Maven?**
    Через тег `<version>` в зависимости. Можно использовать `<dependencyManagement>` в родительском POM для централизованного управления версиями.

16. **Что такое "SNAPSHOT" версии в Maven и как они используются?**
    SNAPSHOT означает, что версия находится в разработке и нестабильна. Maven проверяет обновления SNAPSHOT-версий при каждой сборке (или по расписанию), в отличие от релизных версий.

17. **Как использовать Maven для создания отчета о качестве кода?**
    Подключить плагины отчетности (например, `maven-checkstyle-plugin`, `jacoco-maven-plugin`) в секцию `<reporting>` или `<build>` и запустить фазу `site`.

18. **Какие команды Maven используются для очистки проекта, сборки, тестирования и установки?**
    - `mvn clean` — очистка.
    - `mvn compile` — компиляция.
    - `mvn test` — тестирование.
    - `mvn package` — упаковка.
    - `mvn install` — установка в локальный репозиторий.

19. **Как интегрировать Maven с системой контроля версий, такой как Git?**
    Добавить `target/` и `*.iml` / `.idea/` в файл `.gitignore`. Maven сам не управляет Git, но хорошо с ним взаимодействует (есть `maven-release-plugin`).

20. **Как добавить и настроить сторонний репозиторий в проекте Maven?**
    Добавить секцию `<repositories>` в `pom.xml` и описать репозиторий внутри тега `<repository>`, указав `id` и `url`.
